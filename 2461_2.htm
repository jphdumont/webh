Adding In-Page Google Analytics to Track Specific Page Events

Backend Settings for Site Analytics vs. Page Analytics

  • Adding Analytics to the backend of the site tracks the site globally.
  • Adding Analytics to the page tracks the page level events.

To use Analytics to track your entire site or to apply the tracking code per page paste your Analytics code snippet into the Backend Settings > Analytics for the site. To access this feature in Site Manager go to Manage Sites and select the Backend Settings for a site. Then select the Analytics tab and the code snippet from your Analytics provider.
See Also: Adding Analytics (for the entire site or per page)

Overview of the Adding In-Page Analytics:

  1. Select the Add Provider button to add the provider.
  2. Select Edit to add the code snippet from the Analytics Provider
  3. Add the Analytics provider's code snippet with the Tracking ID to the specific Page Analytics tab.
  4. Add the additional code snippet to include the events for the page using the data-tag-attributes.
  5. In Google Analytics activate the in-page tracking in the reporting.

Step 1: Add an Analytics Provider to the Site

To Use Analytics to track specific data elements on a Page using In-Page Analytics

In-Page Analytics works by tracking the data types on the page. To add analytics for specific page events, for any provider that supports event tracking by link or form submit on a page is called In-Page Analytics.

To access this feature use the page specific analytics tab:

  1. In Site Manager go to Content and select Sections & Pages from the left navigation.
  2. Select the Section of pages, and then the individual page to Edit.
  3. Then on the Analytics tab for that page add the analytics code, this applies to the individual page.

Note. What is specified for In-Page Analytics will over ride what is in the Backend (global) Settings. In this example, Google Analytics is demonstrated. If your provider supports In-Page Analytics then this featured can be used for tracking the In-Page data types, with slight code variations depending on your provider.

Add Provider. Select to add a provider name to the list. Enter the provider name and save.

Configuration. Select the on or off radio button to turn on or off the tracking code setup for this provider.

Edit. Select to edit the icon to add the code snippet from the Analytics provider and define the data-tag-attribute to track a data element on a page. The tracking code entered here will override the Backend Settings Analytics.

Save Settings. Select to save settings.

Step 4: In your Google Analytics Account Enable enhanced link attribution in the reports

After you update your tracking code for in page analytics, the next step is to enable enhanced link attribution in your property settings:

In the Google Analytics account:

  1. Select Admin at the top of any Google Analytics page.
  2. Navigate to the property for which you want to use enhanced link attribution, and click the Property Settings tab.
  3. In the In-Page Analytics section, select the Use enhanced link attribution option.
  4. Select Apply.

Step 5: View In-Page Analytics in the Google Reports

In the Google Analytics account you can access the in page analytics reporting in 2 ways:

  1. Select Reporting from the top of the page, then on the left navigation select Content, click In-Page Analytics. This option opens the report on the page you identified in your profile settings.
  2. Or on the Pages report select (Standard Reporting > Content > Site Content > Pages), when you drill in to a specific page, select the In-Page tab. This option opens the report for that page.
  3. Once the page is open in Google Analytics select a button on the page and the page refreshes with the tracking information for that button (data-tag-attribute).
