Define a Landing Page for a Logged in User

In a Mobile website, you can define a custom link for the profile button. This page is where the user will start once the user logs in. To do this, define the Navigation Value of the msm.profileLoginPage.loginBtn. This path should start with "/tmpl" and then specify the actual path to the page.

How to Define a Custom Link for the Profile Button:

  1. In the MCP Site Manager > go to the Content tab > and select the MOBILE Site to edit > select Sections & Pages > select Account Pages (Mobile) > select the Components Configuration icon for Profile Login page.
  2. Find the msm.profileLoginPage.loginBtn Component ID and modify the Navigation value. The value should be a page within the MOBILE Site that doesn't need input from a previous page and it should start with "/tmpl/...", for example: /tmpl/msm/homePage.xhtml, /tmpl/msm/prof/profilePage.xhtml

To Verify in the Mobile Website:

  1. Select the Profile menu button from the Home Page to go to the Login page.
  2. Verify that upon successful login, the landing page can now be controlled by "Navigation" value for "msm.profileLoginPage.loginBtn" component and is the page just defined in the above step.

Note. The page defined should not require any input from a previous page.
