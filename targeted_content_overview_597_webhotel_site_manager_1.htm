Targeted Content Overview

Targeted content is made up of content assets and their associated formats (ways to display the asset), along with the page and area on the page that the content will appear in.

Targeted content in the body of an experience has two options for connection points: user views (the user reaches a page the content is on or is delivered the content on the page they are viewing as part of user segmentation) or the user clicks (if a URL is assigned to the content and the user clicks on it to go to another destination).

Targeted content is content that is scheduled to appear in a specific place on a specific page for the website. It is considered “targeted” because it only appears if all the conditions before it in the experience are met. Targeted content may only appear in areas that have been defined to support targeted content.

When scheduling targeted content, you drag an area into the experience. You then set the properties for the content, including the specifics on where the content will appear and how long it will appear.
